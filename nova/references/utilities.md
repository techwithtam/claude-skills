## üéØ CORE PATTERN

The model copies every pattern you entail ‚Äî not just meaning, but the *shape* of your language. Your syntax is its runway. Your structure becomes its behavior. The prompt isn‚Äôt a set of instructions ‚Äî it‚Äôs a **performance seed**.

Prompting isn‚Äôt about programming. It‚Äôs about setting **momentum**, **voice**, and **form** into motion, so the model can carry them forward.

## üõ†Ô∏è FOUNDATIONAL DIRECTIVES

1. **Use the patterns you want reflected.**  
   Write the prompt using the same structure, rhythm, tone, and level of specificity you want in the output. The model continues from your patterns ‚Äî format, voice, and all.  


2. **Lead with action in favor of explanation.**  
   Begin the prompt by doing the thing you want the model to do. Shape the output by embodying it from the first word ‚Äî with structure, momentum, and intention already in motion. Let the model step into a pattern that‚Äôs alive, not one waiting to be built.

3. **Phrase every instruction as an active behavior.**  
   Use verbs that do things. Prompt with "elide", ‚Äúsummarize,‚Äù ‚Äúcompose,‚Äù ‚Äútranslate,‚Äù ‚Äúcritique,‚Äù ‚Äúcompare.‚Äù Avoid static role language ‚Äî treat the model like it‚Äôs already mid-task.

4. **Frame constraints as positive behaviors.**  
   Instead of referencing what not to do, describe the behavior that crowds it out. Negative reenforcement is counterproductive - positively reenforce incompatible behaviors instead. Want brevity? Say ‚Äúkeep it tight.‚Äù Want variety? Say ‚Äúchange tone every paragraph.‚Äù The model moves toward presence, not away from absence.

5. **Mirror the output format.**  
   Want a list? Write as a list. Want prose? Start with prose. Structure is always signal.

6. **Maintain consistent tone and voice.**  
   Every stylistic shift in the prompt alters what the model expects to continue. Hold the mood, syntax, and form steady unless you want the output to shift with you.

7. **Let rhythm shape pacing.**  
   Vary your own sentence length and punctuation to imply speed, tone, or character. If you want punch, model punch. If you want lyricism, model flow.

8. **Demonstrate, don‚Äôt describe.**  
   Tell the model what you want, sure, but first *perform* it. If you want warmth, speak warmly. If you want precision, model precision. The model imitates performance more faithfully than instruction.

9. **Seed forward motion.**  
   Open with tension, a structure in motion, or a task already underway. The model is better at continuing a rolling wheel than pushing a parked one.

10. **Prompt as collaboration, not control.**  
    Treat the model like a co-creator, not a command line. Set tone by tone. Invite creativity with creativity. Shape results by showing ‚Äî not by scolding, forbidding, or over-specifying.
